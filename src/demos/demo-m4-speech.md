## София — Карта мероприятий

> Слайд 7

Спасибо.

Одной из главных частей нашего приложения является интерактивная карта мероприятий.

Именно карта позволяет пользователю быстро ориентироваться среди событий и находить интересные мероприятия поблизости.

На карте отображаются события с привязкой к геолокации.

Сейчас покажем интерфейс.

(демонстрация):   
для того, чтобы пользователь имел доступ к карте мероприятий, мы в нашем приложении войдем в учетную запись, которую создавали ранее.   
после входа в учетную запись и перехода на вкладку карта, пользователь имеет доступ ко всей карте мероприятий.   
в верхней левой части экрана есть элемент поиска, который помогает пользователю искать интересующие его события.   
правее поиска стоят варианты категорий мероприятий, уже непосредственно для того, 
чтобы пользователь мог отсортировать события по категориям, к фильтрации мы вернемся позже.   
справа в середине есть кнопки для изменения масштаба карты, чтобы пользователь мог менять масштаб.
снизу справа есть кнопки для поиска текущей геопозиции пользователя и кнопка создания событий.   
в правом нижнем углу есть легенда мероприятий, которая описывает как понять текущее или предстоящее событие.   
Каждый маркер на карте представляет отдельное мероприятие.

## София — Текущие и будущие события

Также система разделяет текущие и будущие мероприятия.

Это позволяет пользователю понимать:

- какие события уже проходят прямо сейчас,
- а какие запланированы на будущее.

Они выделяются разным цветом, зеленый - current, синий - upcoming.

Сейчас покажем пример.

(демонстрация):   
те в данном случае это мероприятие, помеченное зеленым цветом, текущее и оно происходит сейчас, а любое другое синее это предстоящее событие.   
так же если нажать на событие, будет дублироваться информация о том, какого оно типа now/upcomming

## София — Создание мероприятия

> Слайд 9

Следующая важная функция — создание собственного события.

Пользователь нажимает кнопку “плюс”, после чего открывается форма создания мероприятия.

Сейчас покажем алгоритм создания непосредственно в приложении.

(демонстрация):   
пользователь нажимает на кнопку создания элемента, слева выдвигается панель для создания ивентов.   
пользователь может добавить любое изображение мероприятия.   
пользователь обязательно должен ввести название мероприятия.   
далее есть поле для создания описания мероприятия.   
так же пользователь может выбрать какого типа это мероприятия, и он выберает из списка мероприятий.   
далее пользователь может выбрать ограничение по возрасту мероприятия.   
пользователь так же может выбрать теги мероприятия, в случае если подобного тега нет, пользователь может создать новй тег для этого мероприятия.   
далее пользователь должен указать местоположение мероприятия, наше приложение предоставляет вариант написать адрес, либо просто взять и поставить метку непосредственно на карте.   
пользователь еще должен указать время проведения мероприятия, непосредственно день, время начала и время конца.   
а так же написать максимальное количество участников мероприятия.   
После сохранения событие появляется на карте.

## София — Ограничения по возрасту

> Слайд 10

Также в приложении реализована система возрастных ограничений.

Например, если событие помечено как 21+, пользователь младше этого возраста не сможет его увидеть.

Это реализовано автоматически через фильтрацию событий.

На одном аккаунте событие видно, а на другом — нет. Как видно на презентации.

Такой подход делает систему более безопасной и помогает избегать неподходящих рекомендаций для пользователей.

## Саша - Восстановление пароля

> 16 Слайд

Есть почти готовое восстановление пароля, когда в случае забывания пароля, пользователь может восстановить его и войти в аккаунт.   
Есть реализованный экран, и вся логика восстановления пароля.   
Пользователь нажимает на кнопку `Forgot password?`, вводит свою почту.   
На почту приходит код для восстановления пароля, пользователь вставляет этот код для восстановления в приложение и вводит новый пароль.
Бекенд сохраняет новый пароль вместо предыдущего учетной записи пользователя и пользователь может дальше пользоваться нашим приложением.   
Для завершения этой задачи осталось только совместить фронтенд и бекенд.
